## 🌈 1. Rainbow Me — Плавная радужная анимация

**Файл:** `src/game/client/components/system_visuals.cpp`  
**Функция:** `CSystemVisuals::ApplyRainbowColors()`

### Что изменили

**Раньше** цвет тела персонажа менялся дискретно: каждый кадр к переменной `m_RainbowHue` прибавлялось маленькое значение (`+= 0.001f`), из-за чего на скинах с текстурами цвет выглядел резко и неравномерно.

**Теперь** анимация полностью основана на времени через синусоиды:

```
BodyHue  = время * 0.12          → плавный цикл оттенка тела
FeetHue  = время * 0.12 + 0.33   → ноги сдвинуты на 1/3 цикла (отдельный цвет)

Sat      = 0.80 + 0.20 * sin(t * 1.8)   → насыщенность «дышит»
BodyLit  = 0.52 + 0.10 * sin(t * 2.3)   → яркость тела колышется
FeetLit  = 0.52 + 0.10 * sin(t * 2.3 + 1.0)  → яркость ног со сдвигом
```

### Почему это лучше

| Старый вариант | Новый вариант |
|---|---|
| Один цвет на всё тело | Тело и ноги разных оттенков |
| Линейная скорость смены | Синусоидальная — ускоряется и замедляется |
| Тусклые цвета на тёмных скинах | Насыщенность тоже анимирована |
| Зависимость от FPS | Независимость от FPS (время) |

## 🟣 2. Тёмно-фиолетовый дизайн HUD

**Файл:** `src/game/client/components/system_visuals.cpp`  
**Функции:** `RenderWaterMark()`, `RenderFunctionList()`, `RenderSpeedTab()`

### Что изменили

Все три панели HUD получили единый **тёмно-фиолетовый стиль** вместо старого «Liquid Glass» (который хаотично менял цвета):

#### Цветовая палитра
```
Фон         → rgba(0.07, 0.03, 0.14, 0.82)   — тёмный сине-фиолетовый
Рамка       → rgba(0.52, 0.18, 0.88, ~пульс)  — яркий фиолет, мигает синусоидой
Свечение    → rgba(0.45, 0.12, 0.78, 0.18)   — мягкое внешнее свечение
Акцент низа → rgba(0.32, 0.08, 0.60, ~пульс)  — нижняя половина тонирована
Блик        → rgba(0.75, 0.55, 1.00, 0.10)   — световая полоска сверху
```

#### Пульсирующая рамка
Рамка мигает плавно через синус:
```
PulseA = 0.55 + 0.15 * sin(время * 1.4)   → основная рамка
PulseB = 0.35 + 0.10 * sin(время * 1.1)   → нижний акцент
```

#### Надпись WaterMark
- Заголовок: `"system client"` — светло-сиреневый
- Подзаголовок: `"Dark Violet"` — средний фиолет
- Статус: `"ACTIVE"` — цвет рамки (яркий фиолет)

#### Function List
- При выключенном Rainbow: заголовок и пункты в фиолетовых оттенках
- При включённом Rainbow: полный спектр переопределяет фиолет (поведение сохранено)

#### Speed Tab
- Та же тёмная подложка + фиолетовая рамка с пульсом
- Текст скорости — светло-сиреневый (`0.96, 0.90, 1.0`)

## 🤖 3. Dummy Manager — управление дамми

**Файлы:**
- `src/game/client/components/system_visuals.cpp` — логика бинда
- `src/game/client/components/system_visuals.h` — объявление метода
- `src/game/client/components/menus_settings.cpp` — интерфейс
- `src/engine/shared/config_variables.h` — новые переменные конфига

### 3.1 Бинд для свапа (`HandleDummySwapBind`)

Новый метод `HandleDummySwapBind()` вызывается каждый кадр в `OnUpdate()`.

**Как работает:**
1. Читает имя клавиши из конфига `sys_dummy_swap_bind`
2. Перебирает все клавиши клавиатуры и ищет совпадение с именем
3. Детектирует **нажатие по переднему фронту** (rising edge): срабатывает только в момент нажатия, не удержания
4. При нажатии переключает `g_Config.m_ClDummy` (0 = управляем основным, 1 = управляем дамми)

```
Клавиша нажата + раньше не была нажата  →  ClDummy XOR 1
                                             (меняет 0→1 или 1→0)
```

**Переменная конфига:**
```
sys_dummy_swap_bind  — строка с именем клавиши (например "f1")
```

### 3.2 Копирование движений

В меню появилась галочка **"Copy Movement"**, привязанная к уже существующей переменной `cl_dummy_copy_moves`. При включении дамми повторяет все движения основного персонажа (DDNet делает это сам через существующий механизм).

### 3.3 Копирование чата (`SysDummyCopyChat`)

**Файл:** `src/game/client/components/chat.cpp`  
**Функция:** `CChat::SendChat()`

Когда опция включена, после отправки сообщения основным персонажем, то же сообщение отправляется и от дамми через `CMsgPacker`:

```cpp
CMsgPacker MsgDummy(NETMSGTYPE_CL_SAY, false);
MsgDummy.AddInt(Team);             // тип чата (общий/командный)
MsgDummy.AddString(pLine, 256);    // текст сообщения
Client()->SendMsg(!ClDummy, ...);  // отправляем от другого клиента
```

**Переменная конфига:** `sys_dummy_copy_chat` (0/1)

### 3.4 Копирование эмоутов (`SysDummyCopyEmotes`)

**Файл:** `src/game/client/components/emoticon.cpp`  
**Функция:** `CEmoticon::Emote()`

Раньше эмоуты дублировались только если была включена `cl_dummy_copy_moves`. Теперь добавлена отдельная опция:

```cpp
if(g_Config.m_ClDummyCopyMoves || g_Config.m_SysDummyCopyEmotes)
{
    // отправить тот же эмоут от дамми
}
```

**Переменная конфига:** `sys_dummy_copy_emotes` (0/1)

## 🔢 4. Version Spoofer — подмена версии клиента

**Файл:** `src/game/client/components/menus_settings.cpp`  
**Расположение в UI:** Settings → System → Dummy (нижняя часть)

### Что делает

Позволяет клиенту **показывать серверу другую строку версии**. Это может использоваться для тестирования совместимости или маскировки клиента.

> ⚠️ Функция добавляет UI и переменные конфига. Фактическая подстановка строки в сетевой пакет зависит от реализации на стороне движка.

### Пресеты

| Пресет | Строка версии |
|--------|--------------|
| Off | *(не применяется)* |
| TClient 1.5 | `TClient 1.5` |
| Cactus 0.7.5 | `Cactus 0.7.5` |
| DDNet 18.9 | `DDNet 18.9` |
| Custom | *(пользовательский ввод)* |

### Как работает

- При изменении пресета строка обновляется **один раз** (не каждый кадр) — оптимизация
- В режиме «Custom» поле ввода активно и редактируемо
- В режиме любого пресета поле показывает текущее значение (только для чтения)

### Переменные конфига

```
sys_version_spoof_enabled  — включён ли спуфинг (0/1)
sys_version_spoof_preset   — выбранный пресет (0–4)
sys_version_spoof_str      — итоговая строка версии
```

## 🎯 5. Fake Aim Visual — визуальный ложный прицел

**Файл:** `src/game/client/components/menus_settings.cpp`  
**Вкладка:** Settings → System → Misc

### Что изменили

Кнопка была переименована: `"Fake aim(visual)"` → `"Fake Aim Visual (local only)"`

Добавлена **информационная подпись** фиолетовым цветом при включённой опции:

```
Aim visuals shown locally only — other players are not affected.
(Визуальный прицел виден только вам — другие игроки не затронуты.)
```

Это важное уточнение: опция изменяет только то, что видит сам игрок на экране, и не влияет на сетевые данные.
